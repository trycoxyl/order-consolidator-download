Features

We understand warehouse teams need clear daily tools, not extra admin work.

Order Consolidator is built around practical warehouse workflows: importing marketplace picking lists, checking AWB coverage, reviewing courier patterns, and preparing packing allocation summaries.

SKU

Picking list parser

Imports supported marketplace picking list files and turns them into structured order rows for review. The goal is to reduce manual copying before warehouse work starts.

AWB matching and exception review

Helps compare orders against tracking information, then separates matched, unmatched, and missing records so the team can review exceptions before packing.

15PrefixSuffix

Courier pattern review

Shows courier recognition patterns and supports sample-based suggestions for recurring tracking number shapes. This is useful when one-off manual fixes are not enough.

Packing allocation summaries

Produces allocation views that help supervisors compare packing workload. It is a planning aid for repeated warehouse batches, not a replacement for supervisor judgement.

Export and audit-friendly outputs

Keeps useful batch context such as platform, tracking-only rows, pick points, complexity, and allocation fields available for review or exported records.

AdminWarehouse

Local access controls

Supports local login, role separation, and company-scoped visibility for teams that need basic access control without depending on an online login service.

Shared config sync

Provides shared-folder configuration workflows for access data, with manual sync and publish actions for teams using shared folders such as Google Drive Desktop, OneDrive, NAS, or LAN locations.

ZIP

Manual update flow

Releases are distributed as a Windows ZIP. The download page reads the latest release link automatically, while operators can still use the manual GitHub Releases page when needed.
